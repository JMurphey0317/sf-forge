1. Installation

  1. Download or clone this repository to your local machine.
  2. Open Chrome and navigate to chrome://extensions.
  3. Toggle Developer mode on (top-right corner).
  4. Click Load unpacked — select the sf-forge folder (the one containing manifest.json).
  5. Pin the SF Forge icon to the Chrome toolbar for quick access.
  6. Refresh any open Salesforce tabs so content scripts are injected.
Updating: Never remove and re-add the extension. Replace the folder contents and click the ↻ refresh icon at chrome://extensions. The extension ID and all settings stay intact.

2. Connecting to an Org

Every SF Forge tool targets a single active org, shown in the colored lock bar below the topbar. Set it using one of three methods.

Method A — Tab detection Fastest

  1. Open Salesforce in any Chrome tab and log in normally.
  2. Open SF Forge → click Detect Open Tabs on the Dashboard.
  3. Click Use Org on the detected org tile.

Method B — Credential login Persistent

  1. Go to Connect Org in the nav.
  2. Select Sandbox or Production, enter Username, Password, and Security Token.
  3. Set an Alias and Color Tag (amber = sandbox, red = production).
  4. Optionally check Remember credentials for silent reconnect on expiry.
  5. Click Connect & Save Org.

Method C — Session Recovery When expired

  1. Go to Dashboard → Session Recovery.
  2. Click Extract SID from Open Tab — reads the sid cookie from any open Salesforce tab.
  3. Or click Paste SID Manually and enter a Session ID + Instance URL directly.

4. Inspector & SOQL Runner

The most-used tool in SF Forge. Full SOQL runner with history, visual query builder, column sorting, record detail, and an Object & Field Browser.

SOQL Runner features

  • History dropdown — last 20 queries per org, restored on click
  • Save as — name any query to persist it in history permanently
  • Column sort — click any column header to sort ascending/descending
  • Record detail panel — click any result row for a full field/value drawer with "Open in Salesforce" link
  • Load All Pages — paginates up to 10,000 records with live progress
  • Download CSV — exports the current result set

Visual Query Builder

  1. Click Visual Builder above the SOQL textarea.
  2. Select an object — field list loads automatically.
  3. Check the fields you want (Id and Name pre-checked).
  4. Optionally add WHERE, ORDER BY, and LIMIT.
  5. Click Apply to Query — the SOQL appears in the textarea ready to run.

5. Metadata Studio

Search Apex classes, triggers, LWC/Aura bundles, flows, permission sets, and more via Tooling API. Edit source inline and run Apex tests without leaving the extension.

Apex Test Runner v5.1.0

  1. Select ApexClass, search for a class.
  2. Click Run Tests on any row.
  3. Per-method pass/fail/skip table renders with timing, stack traces for failures, and coverage %.

Edit & Save

  1. Search for an ApexClass or ApexTrigger.
  2. Click Edit — source loads into the inline editor.
  3. Make changes → click Save to Org. SF Forge PATCHes the body via Tooling API.

7. Flow Analyzer

Browse all org flows with filters, inspect version history, and activate specific versions directly from SF Forge.

  1. Click Load Flows.
  2. Filter by name, object keyword, or process type.
  3. Click Inspect on any flow to open the version history panel.
  4. Click Activate on any non-active version — SF Forge PATCHes { Status: "Active" } via Tooling API.
Confirmation required: Activation shows a confirm dialog. This action cannot be undone without activating a different version.

8. LWC Lens

  • Enable LWC Lens — injects a hover overlay on the active Salesforce Lightning tab. Hovering any component shows its tag name, API name, and DOM path. Right-click copies the tag name.
  • Source Peek — enter a DeveloperName, SF Forge fetches the bundle and shows all files (JS, HTML, CSS, meta) in tabbed panels via Tooling API.
Tip: Use Source Peek to inspect any LWC — including managed-package components — without needing org access to the component metadata in Setup.

9. Debug Logs

  • Load Recent Logs — shows last 50 logs with user, operation, duration, size, and status
  • View — fetches the full log body with color-coded severity lines
  • Severity filter — All / Errors only / Warnings+ / Debug+ / Flow+ / SOQL/DML only
  • Copy Exception Summary — extracts up to 80 error/exception lines to clipboard
  • Download — saves the raw log file
Workflow: Set a trace flag in Trace Flag Manager → run your code → come back here and click Load Recent Logs.

10. Trace Flag Manager

Create and manage Apex debug trace flags without leaving SF Forge.

  1. Find the User ID in Inspector: SELECT Id, Name FROM User WHERE Name = 'Your Name'
  2. Go to Trace Flag Manager → paste the User Id into Traced Entity.
  3. Set expiration minutes and log levels for each channel.
  4. Click Create Trace Flag. SF Forge creates a DebugLevel then the TraceFlag.
  5. Run your code → go to Debug Logs → Load Recent Logs.

11. Permission Inspector

  • Enter comma-separated Profile/Permission Set names and an Object API name
  • Compare Object Access — Read/Create/Edit/Delete/ViewAll/ModifyAll per profile side-by-side
  • Compare FLS — Read/Edit per field for all matched permission sets, rendered as a matrix grid
  • Export to CSV for audit trail

12. Org Diff

Compare metadata between the active org and any stored org profile.

  • Run Diff — compares ApexClass, CustomObject, FlowDefinitionView, or PermissionSet by name. Shows presence in each org and which side has newer modifications.
  • Field-level Diff — enter an object API name to diff its field arrays between the two orgs. Shows added, removed, and type/length-changed fields.
Requirement: Both orgs must be in the Stored Org Vault (Connect Org page).

13. Deployment Assistant

Generate package.xml and destructiveChanges.xml for use with Salesforce CLI or Metadata API. SF Forge does not deploy directly.

sf project deploy start --manifest package.xml --target-org YourOrgAlias

14. REST Explorer

Raw API request builder. Select GET/POST/PATCH/DELETE, enter a path (e.g. /limits or /sobjects/Account/describe), add a JSON body for POST/PATCH, press Send or Ctrl+Enter. Download the response as JSON.

16. Apex Job Monitor

Monitor queued, running, and recent AsyncApexJob (batch, future, queueable, scheduled) and CronTrigger scheduled jobs. Auto-refreshes every 30 seconds.

  • Filter by job type and status
  • Abort — available for Queued, Processing, and Holding jobs
  • Delete — removes scheduled CronTrigger jobs
  • Export to CSV for reporting

18. Auto-Update Setup

SF Forge checks your GitHub repo for new releases every 6 hours and on every Chrome startup.

One-time configuration

  1. Open SF Forge → Theme Engine → scroll to Update Settings.
  2. Enter your GitHub Owner (username or org) and Repository Name.
  3. Click Save & Check Now.

Publishing a release

  1. Bump "version" in manifest.json and update CHANGELOG.md.
  2. Zip: zip -r sf-forge-vX.Y.Z.zip sf-forge/ -x "*.git*"
  3. Create a GitHub Release tagged vX.Y.Z, attach the .zip as a release asset.
  4. SF Forge detects the update within 6 hours and shows the download banner.

Applying an update

  1. Click Download update in the banner — Chrome downloads the zip.
  2. Unzip and replace the sf-forge folder contents.
  3. Go to chrome://extensions → click ↻ on SF Forge.
Do not remove the extension during updates. Removing and re-adding creates a new extension ID and loses all stored org profiles and settings.
